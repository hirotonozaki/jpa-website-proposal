Project Overview

プロジェクト概要

ペンマンシップ（英字の書法・装飾文字のアート）の普及活動を行う団体「日本ペンマンシップ協会」の公式サイトです。協会紹介・6書体のギャラリー・見学／入会の問い合わせ・会員専用エリアまでを含む、全13ページのコーポレートサイトとして公開・運用されています。

担当範囲

  • 全ページの HTML / CSS コーディング
  • JavaScript による UI 実装（メニュー・スライダー・アニメーション等）
  • PC / タブレット / スマートフォンへのレスポンシブ対応

※ デザイン方針・原稿・写真は協会側の素材をもとに、フロントエンド実装を担当。

Background & Issues

制作背景・課題

ペンマンシップは日本では認知の低い専門分野で、協会にとっては「活動を知ってもらい、見学・入会につなげる」ことが目的になります。サイトには、初めて触れる人にも魅力が伝わり、迷わず情報へたどり着ける設計が求められました。

01

専門用語が多く離脱されやすい

前提知識のない訪問者でも読み進められるよう、情報を段階的に見せる必要がある。

02

6書体という情報量の多さ

特徴の異なる書体を、迷わせず・比較しやすく見せる構成が必要。

03

見学・入会への導線確保

どのページからでも問い合わせへたどり着ける回遊性が必要。

04

端末差による表示崩れ

PCでの閲覧を中心に想定しつつ、スマホでもレイアウトを破綻させない。

私の役割：これらの課題を満たすUIを、表示崩れなく・保守しやすい形でコーディングすること。

Sitemap & Information Architecture

サイトマップ & 情報設計

サイトマップ

TOP（index）
  • About Us（協会について）
  • Penmanship（ペンマンシップ）
  • Gallery（書体・作品）
  • Contact（問い合わせ）
  • Members Only（会員専用）
  • Privacy Policy

Penmanship 配下に 6 書体の詳細ページ（Business Writing / Roundhand / Ornamental / German Text / Flourishing / Engrossing）

情報設計の意図

  • トップを「目次」と位置づけ、4テーマへカード型に分岐させて全体像を把握しやすく。
  • ペンマンシップのページは「基礎 → 書体 → 筆記具 → 学習 → 歴史」と、理解が進む順に整理。
  • グローバルナビ・フッターナビ・トップのカードと、複数の導線で目的のページへ到達できるよう回遊性を確保。
  • 会員向け情報は専用エリアに分離し、一般訪問者の動線をシンプルに保つ。

Design Concept

デザインコンセプト

“インクと紙の上品さ”手書き文字の美しさが主役。装飾は控えめにし、余白で見せる。
フォント
英字見出しに細身でクラシックな Marcellus、和文に Noto Sans / Serif JP を使い分け。FONTPLUS で指定フォントを表示。
配色・余白
落ち着いた背景に濃色テキスト。作品画像が引き立つよう、彩度を抑え余白を広めに。
あしらい
羽根・葉などの SVG 装飾を要所に配置。過剰にならない範囲で世界観を補強。

※ デザイン方針に沿って、各ブラウザ・端末で表示が崩れないよう忠実に実装することを担当した。

Key Points

工夫した点

1

情報を段階的に見せる

スクロール位置を監視し、要素が画面に入ったタイミングで順に表示。情報量の多いページでも一度に見せず、視線を上から誘導して離脱を抑えた。

2

6書体ページの共通化

詳細6ページを共通テンプレートで構築。見た目の統一と、修正時の保守性を両立させた。

3

CSS設計の整理

全ページ共通のヘッダー・フッター・ボタンは common.css に集約。block__element 形式の命名で、どの要素のスタイルか追いやすくした。

4

端末ごとの最適化

スライダーの表示枚数をブレークポイントごとに変更（PC3→タブ2→スマホ1）。どの画面でも見やすさを保った。

Responsive Design

レスポンシブ対応

PC・タブレット・スマートフォンの表示に対応。主なブレークポイントは 1024px / 768px / 500px で、画面幅に応じてレイアウト・ナビ・スライダーの見せ方を切り替えています。

PC
1025px〜
ナビ横並び／スライダー3枚／2カラム中心
TABLET
768〜1024px
余白調整／スライダー2枚／一部1カラム化
SP
〜500px
ハンバーガー／スライダー1枚／縦積み

制作・反省点： 本案件は PC 基準（max-width）で組んだため、次は モバイルファースト＋ブレークポイントの統一 で設計したい。

Improvements & Learnings

改善提案 & 制作を通じて学んだこと

今後の改善提案

  • 画像最適化（WebP化・遅延読み込み）で初回表示を軽量化
  • jQuery 依存からバニラJS / IntersectionObserver への置き換え
  • alt の整備などアクセシビリティの底上げ
  • フォームの入力チェック・送信時フィードバックのUX強化

制作を通じて学んだこと

  • 公開・運用される前提での、表示崩れや端末差への配慮の重要さ。
  • 多ページサイトでの共通化・命名ルールなど、保守性を意識した実装。
  • 「何を作るか」だけでなく「なぜそう作るか（目的・導線）」を考える視点。
